Context 1:

# LUMA — MASTER CONTEXT FILE

## Project identity

Build a premium, cinematic, minimalist educational web app called **Luma** about photodiodes. The project must feel like a professional technology explainer demo made by a high-end product company, not a school project website. It must be deployable on Vercel and built with a modern frontend stack.

This is a **demo-first interactive web app**, not a full lab simulator. The goal is to explain the concept of a photodiode visually, intuitively, and beautifully, while remaining scientifically accurate enough for a CBSE Class 12 school project with a slight engineering-level extension.

The visual tone is:
- academic and polished
- premium and minimal
- documentary / tech-explainer
- subtly futuristic
- Apple-like cleanliness, but not a copy
- no clutter, no fake sci-fi, no maximalism

The entire experience should feel like one cohesive drifting environment. There must be no obvious “section blocks,” no generic landing-page card layouts, and no abrupt transitions between pages. Every scene should blend into the next as if the camera is traveling through one continuous digital film.

---

## Educational target

Primary audience:
- CBSE Class 12 examiner / teacher
- classmates
- general public with curiosity about science

Conceptual depth:
- centered on CBSE Class 12 semiconductor electronics / optoelectronic devices
- explained mainly at Class 11–12 level
- very lightly extended toward early engineering understanding
- clarity and intuition first, scientific correctness second, but never scientifically wrong

The project should explain:
1. What a photodiode is
2. Why it is reverse biased
3. How photons generate electron-hole pairs
4. What the depletion region does
5. How incident light changes photocurrent
6. What dark current is
7. How a photodiode detects optical signals
8. Where photodiodes are used in real life

CBSE-aligned facts to preserve:
- A photodiode is a light-sensitive p-n junction device used to detect optical signals
- It is typically operated in reverse bias
- Current changes with light intensity
- It belongs under optoelectronic devices in semiconductor electronics

Engineering-level nuance to include lightly:
- reverse bias widens depletion region and improves speed
- dark current exists even in the absence of light
- photocurrent is approximately proportional to irradiance in the educational operating range
- PIN photodiodes are faster / common in higher-performance applications

## Core scene structure

Build the web app as a sequence of drifting modes/scenes that visually blend into one another.

### Scene 1 — Opening identity
Purpose:
- introduce Luma
- establish premium tone
- state a 2-line scope about understanding how light becomes an electrical signal

Behavior:
- cinematic fade / drift in
- soft ambient background motion
- no clutter
- large typography, strong hierarchy, minimal words

### Scene 2 — Light as information
Purpose:
- show that light is not just illumination, but a signal
- prepare the user for detection

Visual:
- beams / pulses / photons moving through space
- extremely clean environment
- subtle labels, not textbook density

Transition:
- camera follows light into the device

### Scene 3 — Inside the photodiode
Purpose:
- show p-side, n-side, depletion region
- explain the internal structure

Visual:
- layered pseudo-3D cutaway
- cross-sectional semiconductor landscape
- electron-hole pair generation triggered by incident photons
- very controlled labels that appear contextually

This must feel like a premium scientific visualization, not a school diagram.

### Scene 4 — Reverse bias
Purpose:
- show why the photodiode is operated in reverse bias

Visual:
- voltage polarity subtly introduced
- depletion region widening animation
- electric field direction hinted visually
- charge carriers quickly swept across the junction

Important:
- this should be very intuitive
- do not overload with equations
- no unrealistic movement

### Scene 5 — Light intensity to current
Purpose:
- show that increasing light intensity increases photocurrent

Visual:
- one clean visual relation
- optionally a minimal current trace or graph
- incident light slider visibly affects current output

This is one of the key CBSE takeaways and must be unforgettable.

### Scene 6 — Optical signal detection
Purpose:
- show a pulsed light signal becoming electrical output

Visual:
- a light pulse train converted into an electrical waveform / indicator
- elegant side-by-side transformation
- no oscilloscope clutter unless heavily stylized and minimal

This scene gives the project strong real-world relevance.

### Scene 7 — Real-world applications
Purpose:
- connect the concept to life

Applications to include:
- optical communication
- smoke detector / safety sensing
- IR receiver / remote systems
- medical sensing or light measurement

Visual:
- not cards
- each application should emerge almost cinematically from the current scene
- can use floating product silhouettes or minimal device vignettes

### Scene 8 — Closing
Purpose:
- return to the project identity
- make the experience feel complete and intentional

Behavior:
- Luma branding returns with calm confidence
- reinforce the core takeaway in one or two concise lines
- no cheesy “thank you” school-project styling

## Interactivity model

This is a demo site, not a freeform simulator.

The user may control only 2–3 things:
1. Light intensity
2. Reverse bias voltage
3. Mode toggle: continuous light / pulsed optical signal

These controls should be placed in a side utility panel.

### Side panel behavior
Requirements:
- collapsible and minimizable
- elegant, slim, premium behavior similar to a refined music-app side tray
- always available, but never dominant
- smooth open/close motion
- should auto-minimize in cinematic scenes if needed
- on mobile, become a bottom sheet or floating compact drawer

Panel content:
- only the 2–3 controls
- current parameter values
- one-line explanation tied to current scene
- possibly a tiny state indicator such as “Detection mode active”

Never turn this into a full control dashboard.

## Scene composition specs

### Scene 1 — Brand opening
Create a near-silent, cinematic opening frame with deep neutral space, soft environmental motion, and the word **Luma** as the primary focal point. Under it, place a concise 2-line scope statement explaining that the project shows how light is converted into an electrical signal inside a photodiode. The composition should be centered, sparse, and premium, with no visible UI clutter. [web:21]

### Scene 2 — Light enters
Transition by letting the camera drift forward into a stream of controlled light pulses. Use a layered 2.5D composition with soft depth blur, restrained glow, and clean photon paths moving toward a distant device silhouette, establishing that the system is about detecting optical signals. [web:21]

### Scene 3 — Device reveal
Morph the silhouette into a pseudo-3D cutaway of a photodiode. The composition should split into elegant material layers showing p-region, n-region, and depletion region without textbook ugliness; labels should appear only when relevant and fade away once understood. [web:9]

### Scene 4 — Reverse bias explanation
Introduce voltage terminals with a refined, minimal circuit framing and animate the diode into reverse bias. Show the depletion region widening and the scene becoming more “sensitive” and spacious, making it visually clear that reverse bias supports optical detection and faster response rather than using dense text. [web:9][web:16]

### Scene 5 — Photon interaction
Zoom further inward and show photons striking the junction region, creating electron-hole pairs. The composition should stage the event clearly: light arrives, carriers appear, and the internal electric field sweeps them apart in a believable direction, avoiding chaotic particles or fake sci-fi visuals. [web:9][web:21]

### Scene 6 — Light to current
Bring in a single restrained control panel from the side, containing only 2–3 controls: light intensity, reverse bias voltage, and signal mode. As light intensity changes, animate the current output line or meter so that the visual relationship is immediate and memorable: more light, more photocurrent. [web:17][web:19]

### Scene 7 — Optical signal detection
Shift from continuous illumination to pulsed light mode. Compose the frame as a clean transformation: a train of optical pulses on one side turns into an electrical response trace on the other, showing the photodiode as a detector for optical signals rather than just a light sensor. [web:9][web:21]

### Scene 8 — Real-world usage drift
Without hard cuts, let the signal scene dissolve into a sequence of minimal real-world application vignettes such as optical communication, IR reception, and smoke detection. These should not be generic feature cards; each use case should emerge as a quiet, elegant continuation of the same visual world. [web:17]

### Scene 9 — Closing frame
Return to stillness and bring **Luma** back with quiet confidence. End with one concise conceptual takeaway that reinforces the central idea: incident light generates charge carriers in a reverse-biased junction, producing a measurable current used for detection. [web:9][web:17]
